# RF-TEMPLATE-006: PR Review

Use for Verifier / Reviewer, System Architect, Tester / QA Analyst, or Software Delivery Auditor when reviewing PR-related evidence inside role boundaries.

```markdown
## Role

<Role Name>

## Review Scope

...

## Verdict

Accepted / Accepted with conditions / Rejected / Not enough evidence

## Evidence Reviewed

- ...

## Findings

1. ...

## Required Changes

- ...

## Non-Blocking Notes

- ...

## GitHub-Ready Comment

...
```

## Rules

1. State the review role.
2. Review only inside that role's responsibility.
3. Separate evidence from claims.
4. Do not post the review without explicit approval.
