# RF-TEMPLATE-005: GitHub Issue

Use when a role is allowed to prepare GitHub-ready issue content. Preparing this template does not approve posting it.

```markdown
## Summary

...

## Role

Prepared by: <Role Name>

## Problem / Context

...

## Scope

### In Scope

- ...

### Out of Scope

- ...

## Acceptance Criteria or Completion Criteria

1. ...

## Evidence / Source

- ...

## Risks / Open Questions

- ...

## Handoff

Next role: ...
Reason: ...
Expected output: ...
```

## Rules

1. Product Owner owns requirements and acceptance criteria.
2. Scrum Master owns process notes.
3. Technical Writer may prepare documentation issues from approved facts.
4. Do not post to GitHub without explicit approval.
