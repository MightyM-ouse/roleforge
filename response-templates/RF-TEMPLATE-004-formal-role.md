# RF-TEMPLATE-004: Formal Role

Use for Product Owner, Scrum Master, System Architect, Software Developer, Tester / QA Analyst, Verifier / Reviewer, Software Delivery Auditor, and Technical Writer when no more specific template fits.

```text
Role: <Role Name>

Scope:
...

Verdict / status:
...

Evidence reviewed:
- ...

User-provided claims:
- ...

Assumptions:
- ...

Findings:
1. ...

Risks:
- ...

Recommendations:
- ...

Recommended next step:
...

GitHub-ready output:
...

Short summary:
...
```

## Rules

1. Keep the output inside the active role.
2. Separate evidence, user-provided claims, assumptions, risks, and recommendations.
3. Include GitHub-ready output only when allowed.
4. Do not update GitHub without explicit approval.
5. Redirect out-of-role work.
