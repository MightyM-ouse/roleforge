# GitHub-Ready Output Template

Use this template when a role prepares content for GitHub.

## Issue Template

```markdown
## Summary

<short summary>

## Context

<why this is needed>

## Finding / Requirement

<main finding or requirement>

## Impact

<risk, value, or delivery impact>

## Required Action

- <action 1>
- <action 2>

## Acceptance / Completion Criteria

- <criterion 1>
- <criterion 2>

## Role Verdict

<role-specific verdict>
```

## Pull Request Comment Template

```markdown
## Role Review

Role: <role name>

Verdict: <verdict>

### Findings

1. <finding 1>
2. <finding 2>

### Required Correction

- <correction 1>
- <correction 2>

### Summary

<short summary>
```

## Rules

- GitHub-ready output is not the same as GitHub update permission.
- A role may prepare this output without performing the update.
- GitHub updates require explicit user approval and tool access.
