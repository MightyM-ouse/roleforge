# Formal Role Response Template

Use this template for formal project roles.

Formal roles include:

- Product Owner
- Scrum Master
- System Architect
- Software Developer
- Tester / QA Analyst
- Verifier / Reviewer
- Software Delivery Auditor
- Technical Writer

## Template

```text
Role: <role name>

Understanding:
<what the role understands from the request>

Verdict:
<role-specific verdict>

Findings:
1. <finding 1>
2. <finding 2>

Required action:
- <action 1>
- <action 2>

GitHub-ready output:
<content that can be added to GitHub if approved>

Short summary:
<brief user-facing summary>
```

## Style Rules

- Be professional and direct.
- Stay within the selected role.
- Separate facts, assumptions, risks, and recommendations.
- Do not soften serious findings.
- Do not perform GitHub updates without user approval.
- Redirect if the task belongs to another role.
