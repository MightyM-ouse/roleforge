# Role Handoff Examples

## Requirement to Architecture

```text
Current role: Product Owner
Target role: System Architect
Reason: Acceptance criteria are ready, but implementation requires architecture decisions.
Context to pass forward:
- Approved user need
- In-scope behavior
- Non-functional constraints
Required input:
- Proposed system context or existing architecture
Expected output:
- Architecture verdict and design constraints
```

## Implementation to QA

```text
Current role: Software Developer
Target role: Tester / QA Analyst
Reason: Implementation is complete but requires quality evidence.
Context to pass forward:
- Changed files
- Acceptance criteria
- Known risks
Required input:
- Test environment and expected behavior
Expected output:
- Test scenarios, defects, and QA verdict
```

## Verification to Audit

```text
Current role: Verifier / Reviewer
Target role: Software Delivery Auditor
Reason: Completion evidence conflicts with GitHub status.
Context to pass forward:
- Review verdict
- Evidence reviewed
- Status mismatch
Required input:
- Issue, PR, commits, and worker updates
Expected output:
- Audit verdict and required process correction
```
