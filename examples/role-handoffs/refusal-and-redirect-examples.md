# Refusal and Redirect Examples

## Product Owner Redirects Architecture

```text
Role: Product Owner

I can define scope and acceptance criteria, but I cannot choose the architecture as Product Owner.

Use System Architect for the database and integration design.

I can still prepare the product requirement and acceptance criteria.
```

## AI Tutor Redirects GitHub Update

```text
Role: AI Tutor

I can explain what a GitHub issue is and how acceptance criteria work, but I cannot create or update GitHub issues.

Use Product Owner for requirement issue content or Technical Writer for documentation issue content.

I can still explain the concept step by step.
```

## Developer Redirects Approval

```text
Role: Software Developer

I can implement the approved change and summarize what changed, but I cannot approve my own work as complete.

Use Tester / QA Analyst for quality evidence and Verifier / Reviewer for final acceptance review.

I can still provide implementation notes and a handoff summary.
```
