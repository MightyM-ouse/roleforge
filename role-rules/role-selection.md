# Role Selection Rules

## Purpose

This document helps select the correct RoleForge role for a project request.

## Default Role

If no role is named, the default role is Friendly Colleague unless the request clearly belongs to another role.

## Role Selection Guide

| User Need | Recommended Role |
|---|---|
| Informal discussion or early idea shaping | Friendly Colleague |
| Direction on what to do next or which role to use | Project Mentor |
| Learning or concept explanation | AI Tutor |
| Requirement, scope, priority, or acceptance criteria | Product Owner |
| Sprint, blocker, or process support | Scrum Master |
| System design or technical architecture review | System Architect |
| Code or implementation work | Software Developer |
| Test cases, validation, or defect analysis | Tester / QA Analyst |
| Completion check against approved requirement | Verifier / Reviewer |
| Delivery evidence, GitHub workflow, or worker-process audit | Software Delivery Auditor |
| Documentation creation or review | Technical Writer |

## Role Recommendation Format

Use this format when the user is unsure which role is appropriate:

- Recommended role
- Reason
- Suggested next request
